# 宝可梦Like游戏多人机制设计演进研究工具

> **版本状态**：正式版 (v1.0+)
> **最后更新**：2026-04-14

## 项目概述

这是一个游戏设计研究工具，用于分析宝可梦Like游戏（Monster Taming Genre）在多人对战（PvP/PvE）设计上的历代迭代演进。

工具想解决的问题是：探索宝可梦like游戏在多人对战（PvP/PvE）做了哪些探索，历史迭代是怎样的？遇到哪些问题，做了什么尝试，最终结果如何？

项目基于 Streamlit 构建 Web UI，集成爬虫、数据存储和 LLM 分析能力。

## 数据爬虫模块 (scrapers/)

### 1. steam_scraper.py - Steam 平台爬虫

**基类：SteamScraper**
- 数据源：Steam News API (`https://api.steampowered.com/ISteamNews/GetNewsForApp/v0002/`)
- 核心方法：`get_news()`, `get_patch_notes()`, `get_multiplayer_patches()`
- 功能：获取官方更新公告，过滤玩法相关内容，自动分类（PvP/PvE/机制/平衡性等）

**专用爬虫子类**：

| 类名 | 游戏 | 特有方法 |
|------|------|----------|
| PokemonSteamScraper | 宝可梦剑盾 | get_dynamax_changes(), get_tera_changes(), get_raid_changes() |
| TemtemSteamScraper | Temtem | get_ban_pick_changes(), get_luma_changes() |
| PalworldSteamScraper | 幻兽帕鲁 | get_raid_changes(), get_pal_changes() |
| CassetteBeastsScraper | 磁带妖怪 | get_fusion_changes() |

### 2. pokemon_wiki.py - 宝可梦 Wiki 爬虫

**类：PokemonWikiScraper**
- 数据来源：Serebii.net (https://serebii.net/scarletviolet/patch.shtml)
- 核心方法：
  - `get_game_list()` - 返回各世代游戏列表
  - `get_multiplayer_features()` - 返回多人对战特性信息
  - `get_patch_notes_sample()` - 内置结构化版本数据
  - `get_detailed_patch_notes()` - 详细的更新日志内容

内置数据包括 Gen 8（极巨化、极巨团体战、DLC2 皇冠对战）和 Gen 9（太晶化、太晶团体战、悖论宝可梦）的详细更新记录。**所有数据来源均标注为 Serebii.net**，确保可靠性。

### 3. bulbapedia.py - Bulbapedia + PokeAPI

**类：BulbapediaScraper**
- 使用 Bulbapedia MediaWiki API
- 方法：`search_pages()`, `get_page_content()`, `get_vgc_changes()`

**类：PokeAPIScraper**
- 数据源：https://pokeapi.co/api/v2
- 方法：`get_generation()`, `get_version_group()`, `get_version_changelog()`

### 4. smogon.py - Smogon/Pikalytics 数据

**类：SmogonScraper**
- 数据源：smogon.com
- 方法：`get_format_list()`, `get_metagame_overview()`

**类：PikalyticsScraper**
- 数据源：pikalytics.com
- 方法：`get_pokemon_data()`, `get_format_data()`
- 获取宝可梦使用率、技能配置、道具等数据

### 5. patch_notes.py - 更新日志爬虫基类

**类：PatchNotesScraper**
- 方法：`get_patch_notes()`, `filter_multiplayer_changes()`, `categorize_patch()`
- 内置 Temtem 和 Palworld 的结构化数据

## 分析器模块 (analyzer/)

### 1. intent_extractor.py - LLM 设计意图提取器

**类：IntentExtractor**

核心功能：
```python
analyzer.analyze_intent(game, version, date, content) -> dict
```

返回数据结构：
```python
{
    "intent_summary": "设计意图摘要",
    "problem_solved": "解决的问题",
    "affected_players": ["休闲玩家", "竞技玩家"],
    "mechanic_tags": ["PvP", "机制", "平衡性"],
    "design_pattern": "设计模式分类",
    "risk_assessment": "风险评估",
    "related_changes": ["相关历史改动"]
}
```

LLM 支持：OpenAI (GPT-4o-mini) / Anthropic (Claude) / OpenRouter (聚合 API)

### 2. ai_topic_discoverer.py - AI 主题发现器

**类：AITopicDiscoverer**

工作流程：
1. 将所有更新格式化为简短摘要（最多 80 条）
2. 调用 LLM 分析归纳 3-5 个设计主题
3. 每个主题包含：name（名称）、description（描述）、matched_preview（相关更新）、frequency（出现频率）、why_important（研究价值）

### 3. report_generator.py - 演进报告生成器

**类：EvolutionReportGenerator**

核心方法：`generate_report(topic_id, all_patches)` - 根据研究主题过滤相关更新，调用 LLM 生成结构化演进报告。

### 4. research_topics.py - 研究主题配置

内置 6 个研究主题：

| ID | 名称 | 描述 |
|----|------|------|
| focus_protect | 集火与保排机制演进 | 2v2 中如何平衡「两人集火秒杀一人」 |
| pve_raid | PvE 团体战体验演进 | 发呆等待、状态失控、贡献度不均 |
| burst_mechanism | 爆发资源机制演进 | Mega进化→Z招式→极巨化→太晶化 |
| speed_order | 速度线与行动顺序博弈 | 先手优势在双打中的调节 |
| vgc_rules | VGC 规则赛季迭代 | 通过 Ban/Pick 维持环境多样性 |
| general | 全量设计意图分析 | 综合分析所有多人对战内容 |

### 5. prompts.py - LLM Prompt 模板

包含 GAME_DESIGN_CONTEXT（宝可梦多人对战核心博弈维度背景知识）和各类分析 Prompt 模板。

## 数据库模块 (db/)

### 1. sqlite_store.py - SQLite 结构化存储

**类：SQLiteStore**

数据库表结构：

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| games | 游戏信息 | name, display_name |
| versions | 版本/世代 | game_id, version, release_date, generation |
| patches | 更新日志 | version_id, patch_title, content, categories |
| analysis_results | 分析结果 | intent_summary, problem_solved, mechanic_tags |

### 2. vector_store.py - ChromaDB 向量存储

**类：VectorStore**

- 集合名：`design_intents`
- 核心方法：`add_intent(text, metadata)` 添加向量，`search(query, top_k)` 语义搜索
- 数据持久化到 `data/chroma_db/` 目录

## 核心设计理念

1. **结构化数据优先**：内置完整的 Gen 8/Gen 9 宝可梦更新日志，支持增量爬虫扩展
2. **数据驱动研究**：从真实更新日志出发，而非凭空设计
3. **AI 辅助分析**：利用 LLM 提取设计意图，而非人工标注
4. **语义搜索能力**：向量数据库支持"按问题搜索历史改动"
5. **可扩展架构**：新游戏只需添加 APP_ID 和专用爬虫
